1 The Problem

Medical scanners require regular quality assurance. Uniformity testing answers a simple question: "Is this scanner producing consistent images across the entire field?"

flowchart LR A[Uniform Phantom] --> B[Medical Scanner] B --> C[Output Image] C --> D{Uniform?} D -->|Yes| E[PASS] D -->|No| F[FAIL] style A fill:#e6f2ff,stroke:#0066cc style B fill:#f1f5f9,stroke:#64748b style C fill:#f1f5f9,stroke:#64748b style E fill:#d4edda,stroke:#28a745 style F fill:#f8d7da,stroke:#dc3545

A technician places a uniform phantom in the scanner. The resulting image should be uniform. Deviations indicate calibration issues, hardware problems, or artifacts that could affect patient diagnoses.

Our Task

Automate this PASS/FAIL decision with high reliability and interpretability.

2 Our Approach

We approached this as integrators, not researchers. The goal isn't to publish a paper—it's to build something that works, that technicians can trust, and that doesn't require a PhD to maintain.

3 The Baseline: Physics-Informed Features

"Uniformity" has a physical meaning: low variance in pixel intensity. Before using any ML, we asked: what would a human QA specialist look for?

4 The Advanced Model: AI with Interpretability

Healthcare demands explainability. A black-box CNN that says "FAIL" isn't useful if the technician can't understand why.

Prompt Engineering Iteration

v1: Basic Prompt (GPT-4o)

66.7% accuracy — Missed 3 FAILs. Model was looking for "obvious" issues only.

v2: Conservative Prompt (GPT-4o)

77.8% accuracy, 100% recall — Added "when in doubt, FAIL" instruction. Catches all failures.

v3: Latest Model (GPT-5.2)

77.8% accuracy, 100% recall — Upgraded to OpenAI's latest flagship. Richer, more precise reasoning.

Why Conservative?

In medical QA, missing a failure (false negative) is dangerous — a bad scanner stays in use. A false alarm just triggers human review. We tuned for high recall over high precision.

6 Production Considerations

Offline Mode

The assignment asked: "If you use commercial APIs, ensure your solution can also run in an offline/fallback mode."

flowchart LR A[Image] --> B{In Cache?} B -->|Yes| C[Return Cached] B -->|No| D{API Key?} D -->|Yes| E[Call GPT-5.2] D -->|No| F[Fail-Safe: FAIL] E --> G[Cache Response] G --> C style C fill:#d4edda,stroke:#28a745 style F fill:#f8d7da,stroke:#dc3545

Our Solution

All GPT-5.2 responses are cached by image hash. First run makes API calls and saves results. Subsequent runs use cache — no internet required. This also makes results reproducible.

Deployment

# Single command runs entire pipeline
python main.py --data-dir /path/to/images

# Skip GPT-5.2 for faster processing
python main.py --no-advanced

# Verbose output for debugging
python main.py --verbose
